What to Include in Your Report

Describe the bug clearly. What did you see? What went wrong?
1. Open [screen name]
2. Do [specific action]
3. See [what went wrong]
Expected: what should have happened
Actual: what actually happened
Platform: Windows / Android
OS version: (e.g. Windows 11 24H2 / Android 13)
App version: (found in Settings → About)

How to File a GitHub Issue

  1. Click GitHub Issue above — it opens a new issue with the bug report template pre-filled.
  2. You do not need a GitHub account if the repo is set to allow anonymous issues. If prompted to sign in, create a free GitHub account (takes 60 seconds).
  3. Fill in each section: what happened, steps to reproduce, expected vs. actual, and your Windows + app version.
  4. Add a screenshot if possible — drag and drop it directly into the text box.
  5. Click Submit new issue. You'll receive email updates as the bug is reviewed and fixed.

Not sure if it's a bug? Submit it anyway. What seems like user error might reveal a UX problem worth fixing. Every report is valuable — even "I couldn't figure out how to do X."
